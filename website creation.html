How I made this website

I started this journey with watching my teachers videos on the subject that can be found on YouTube as well as downloading software to get this project done. This was my first attempt at using HTML so I thought it wise to start out with a template for my site. Similar templates as well as this one can be found on HTML5up.

The program I used to manipulate the template is called Brackets, a surprisingly simple and easy to use editor. After tinkering for a while in the Brackets editor I got some sense to what í was doing and started thinking about the way I wanted my webpage to look like. I sketched up a little drawing with my idea, the goal was to keep it as easy to navigate as í could while still having it be somewhat elegant.

So now the real work would begin, editing the template to match my vision.

On this picture you can see how the template looked before my changes, if you want to see the original code in its entirety it may be found here, front page, about me. For comparison to the final product, you could right click on this page and click Inspect or see it here by clicking front page, about me. After comparing the stock template and the code for this website you are well on your way with knowing how to make your own.

When I felt happy with my result, I had to get it online for you to see. That is where GitHub and Git comes into play, GitHub is a webpage to manage all kinds of projects. There you have version control over the code you are working on for an example. Git is then used to manage the files you have in your project’s repository. The process of uploading this website started with me creating a repository on GitHub. Instructions on how that is done may be found here.

Then I uploaded my files using Git GUI, the reason I went with that option rather than Git bash was ease of use. I had tried and failed a few times with Git bash and for some reason the bash dialog box was not working as it should have. Uploading with Git GUI was very easy and straight forward, right clicking in the folder where you keep the files you want in your repository and clicking Git GUI opens this window.

There you see some files I the upper left box that have not been uploaded to the repository, by first clicking Stage changed (marked Nr.1), then writing a commit message in the text box (marked Nr.2), click Commit (marked Nr. 3) and finally clicking Push (marked Nr. 4).

Now that everything was finished and ready I made a link for the webpage, that was done under the settings tab on the repository and finding the GitHub Pages settings seen on the picture below. There you select your branch as main and save. Then you get the shareable link for your page.
